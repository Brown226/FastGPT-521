# FastGPT 商业功能显示控制分析报告

> 分析日期: 2026-03-27
> 项目版本: FastGPT 4.2.3

---

## 一、概述

本文档分析 `docs/版本说明.md` 中列出的商业功能的前端显示控制机制，识别哪些功能因为 `licenseData` 未配置而无法显示。

---

## 二、功能显示控制机制分类

### 2.1 按控制方式分类

| 控制方式 | 功能列表 | 说明 |
|----------|----------|------|
| **License 控制** | `show_dataset_enhance`、`show_batch_eval`、`isPlus` | 依赖 `licenseData` |
| **权限系统控制** | `hasReadChatLogPer`（日志看板） | 依赖数据库权限配置 |
| **环境变量控制** | `hideChatCopyrightSetting`、`show_git` | 依赖环境变量 |
| **功能开关** | `pay`、`sso`、`customTemplates` | 依赖 `licenseData.functions` |
| **无显式控制** | 大部分基础功能 | 默认显示 |

## 三、License 相关功能（问题所在）

### 3.1 当前 .env 配置

检查 `projects/app/src/service/common/system/index.ts`:

```typescript
show_dataset_enhance: licenseData?.functions?.datasetEnhance,  // undefined
show_batch_eval: licenseData?.functions?.batchEval,           // undefined
isPlus: !!licenseData,                                        // false
```

### 3.2 问题原因

当前 `PRO_URL` 环境变量**未设置**，导致：

```typescript
if (!FastGPTProUrl) {
  return { fastgptConfig: {} };
}
```

`licenseData` 为 `undefined`，所有依赖它的功能都显示为 `false` 或被隐藏。

### 3.3 受影响的商业功能

| 功能 | 控制字段 | 当前值 | 受影响 |
|------|----------|--------|--------|
| 知识库索引增强 | `show_dataset_enhance` | `undefined` | ✅ 是 |
| 应用评测 | `show_batch_eval` | `undefined` | ✅ 是 |
| 自定义版权信息 | 环境变量 `HIDE_CHAT_COPYRIGHT_SETTING` | `true` | ❌ 否（已隐藏） |

## 四、权限系统控制功能

### 4.1 日志看板（hasReadChatLogPer）

**控制位置**: `RouteTab.tsx`

```typescript
...(appDetail.permission.hasReadChatLogPer
  ? [{ label: t('app:chat_logs'), value: TabEnum.logs }]
  : [])
```

**数据库验证**: root 账号权限为 `4294967295` (OwnerPermissionVal)，**权限正确**。

**问题**: API 响应可能未正确返回权限。需要检查 API `/api/core/app/detail`。

### 4.2 其他权限控制功能

| 功能 | 权限字段 | 说明 |
|------|----------|------|
| 应用编辑 | `hasWritePer` | 正常 |
| 发布渠道管理 | `hasManagePer` | 正常 |
| 查看聊天日志 | `hasReadChatLogPer` | **可能有问题** |

## 七、解决方案

### 7.1 方案一：配置 PRO_URL（推荐）

1. 部署 License 服务或使用模拟服务
2. 在 `.env` 中添加：
   ```bash
   PRO_URL=http://your-license-server
   ```

**效果**: 启用所有 License 相关功能

### 7.2 方案二：环境变量调整

修改 `projects/app/src/service/common/system/index.ts`:

```typescript
// 将 undefined 改为 true，强制启用功能
show_dataset_enhance: true,  // 强制启用
show_batch_eval: true,       // 强制启用
```

### 7.3 方案三：针对日志功能单独处理

修改前端权限检查逻辑，确保 Owner 账号始终显示日志 Tab：

修改 `RouteTab.tsx`:

```typescript
const isOwner = appDetail.permission.isOwner;

const tabList = useMemo(
  () => [
    // ...其他 tabs
    ...(isOwner || appDetail.permission.hasReadChatLogPer
      ? [{ label: t('app:chat_logs'), value: TabEnum.logs }]
      : [])
  ],
  // ...
);
```
